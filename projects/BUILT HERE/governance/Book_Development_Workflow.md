# Book Development Workflow

**Project:** *Built Here: The Cedar Rapids Advantage*  
**Status:** Draft v1.0

## Purpose

This document defines the end-to-end production workflow for the book. It specifies how work progresses, who owns each stage, required artifacts, approval gates, and completion criteria.

---

# Guiding Principle

The workflow exists to produce the best possible book.

Speed is valuable only when it does not reduce accuracy, coherence, or editorial quality.

The workflow is subordinate to `The_Promise_of_Built_Here.md`, `Writing_Constitution_v2.md`, `Chapter_Contract_v2.md`, and `Narrative_Test.md`. Research timelines may verify facts, but chronology may never become the manuscript's narrative structure.

---

# Phase 1 — Project Definition

## Inputs
- Book_Project_Charter.md
- Editorial_Vision.md
- Research_Questions.md
- Evidence_Standard.md

## Outputs
- Approved governing documents
- Active repository
- Initial project state

**Owner:** Project Owner

---

# Phase 2 — Discovery

## Objectives

- Build the research universe
- Register sources
- Identify founders, companies, institutions, timelines, and peer cities
- Test the working thesis

## Primary Artifacts

- Source Records
- Research Notes
- Timeline
- Entity records
- Decision log

**Owner:** Research agents coordinated by Hermes

Exit Criteria:
- Core research questions substantially addressed
- Major evidence gaps identified

---

# Phase 3 — Thesis Assessment

## Objectives

- Evaluate the evidence
- Challenge assumptions
- Refine or revise the working thesis
- Recommend narrative direction

Outputs:
- Thesis Assessment
- Risks
- Counterarguments
- Supporting evidence map

## Human Approval Gate 1

Project Owner approves or redirects the thesis and overall narrative architecture.

---

# Phase 4 — Narrative Architecture

Objectives:

- Build the book structure
- Define parts
- Define chapter purposes
- Sequence the narrative
- Map research questions to chapters

Outputs:

- Narrative architecture
- Table of contents
- Chapter dependency map

## Human Approval Gate 2

Project Owner approves the table of contents.

---

# Phase 5 — Chapter Planning

For each chapter:

Produce:

- Chapter Brief
- Required evidence
- Narrative objective
- Key scenes
- Research dependencies
- One named entrepreneurial protagonist
- Documented business problem, stakes, pivotal decision, consequence, and legacy
- Cedar Rapids context that directly changes or is changed by the protagonist

## Human Approval Gate 3

Approve each Chapter Brief before drafting.

---

# Phase 6 — Drafting

Writing agents create first drafts using only approved evidence. Every draft must tell one entrepreneur's story; history remains supporting material.

Outputs:

- First Draft
- Citation map
- Open questions

## Human Approval Gate 4

Review and approve, reject, or redirect the first draft.

---

# Phase 7 — Editorial Review

Specialist agents perform:

- developmental editing
- structural review
- continuity review
- redundancy detection
- voice consistency
- pacing review
- entrepreneur-centered narrative review
- chapter-opening and page-turning review

Outputs:

- Editorial report
- Revision requests

---

# Phase 8 — Fact Verification

Verification agents:

- confirm citations
- verify chronology
- resolve conflicts
- validate quotations
- check evidence links

Outputs:

- Verification report
- Outstanding issues

A chapter cannot advance until verification passes.

---

# Phase 9 — Final Chapter Approval

Produce revised chapter.

## Human Approval Gate 5

Project Owner approves final chapter.

Approved chapters are locked unless formally reopened.

---

# Phase 10 — Manuscript Assembly

Hermes assembles:

- chapters
- front matter
- acknowledgements
- AI disclosure
- appendices
- bibliography
- index inputs

---

# Phase 11 — Full Book Review

Perform:

- continuity review
- repetition review
- terminology review
- timeline consistency
- thematic consistency
- final editorial pass
- entrepreneur prominence and chapter-to-chapter momentum

Outputs:

- Final issue log
- Publication candidate

---

# Phase 12 — Publication

Deliver:

- publication manuscript
- repository archive
- final evidence archive
- release package

Final authority rests with the Project Owner.

---

# Parallel Work

Hermes may run research, source processing, entity extraction, timeline construction, and evidence registration concurrently where dependencies permit.

Hermes may not bypass approval gates.

---

# Failure Handling

If work fails review:

1. Record deficiency.
2. Assign corrective action.
3. Regenerate only affected artifacts.
4. Preserve prior versions.
5. Resume workflow.

No failed artifact is silently overwritten.

---

# Definition of Complete

The workflow is complete only when:

- governing standards were followed,
- approval gates passed,
- evidence verified,
- manuscript assembled,
- AI disclosure prepared,
- publication approved.

The production pipeline exists to make quality repeatable, not merely faster.
