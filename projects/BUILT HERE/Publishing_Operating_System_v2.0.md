# Purpose

The Publishing Operating System (POS) exists to produce a historically accurate, emotionally compelling book about entrepreneurs whose courageous decisions shaped Cedar Rapids.

The POS is designed around enduring artifacts rather than specific AI models or individual agents.

Agents are replaceable.

Artifacts are the long-lived contracts.

---

# Governing Principle

The objective of this system is not to produce chapters.

The objective is to produce a book that answers one question:

> **Why did generations of entrepreneurs repeatedly choose to build here, and how did those choices create a culture that continued producing builders?**

Every artifact, agent, and workflow exists to serve that purpose.

# Editorial Standard

Every chapter must satisfy five tests.

1. The entrepreneur is memorable.
2. The courageous decision is unmistakable.
3. Cedar Rapids meaningfully shapes the story.
4. The chapter advances the book's central investigation.
5. The reader wants to continue.

Failure of any test returns the chapter for revision.

---

# Evidence Standard

The historical record defines the limits of the narrative.

The system may:

- interpret
- synthesize
- prioritize
- connect

The system may never:

- invent facts
- invent dialogue
- invent motives
- invent private thoughts
- exaggerate unsupported conclusions

---

# Repository Philosophy

The repository stores decisions, not working memory.

Permanent artifacts belong in the repository.

Agent reasoning, candidate rankings, investigative notes, and temporary analysis do not.

The repository preserves what future agents must know—not everything previous agents thought.

---

# Freeze Policy

Architecture changes require a demonstrated weakness in the production pipeline.

Do not add agents merely because they are possible.

Every new artifact or agent must solve a problem that cannot be solved by the existing system.

Default to improving existing components before introducing new ones.

---

# Version 2 Objective

Version 2 is not a revision of Version 1.

It is a complete regeneration from first principles using this Publishing Operating System.

The manuscript should emerge from the architecture—not from incremental edits to previous drafts.

---

# Success Standard

The Publishing Operating System succeeds when:

- readers remember entrepreneurs before dates,
- courage is visible without exaggeration,
- history serves the story,
- every chapter is indispensable,
- and the finished book inspires future builders to create something worthy of the next edition of *Built Here*.
